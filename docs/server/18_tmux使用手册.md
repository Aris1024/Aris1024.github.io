# mux 核心操作速查表
为你整理了一份 **Tmux 核心操作速查表**。建议你可以将这段内容截图或复制保存到备忘录中，忘记的时候随时翻看。

---

### ⭐️ 核心黄金法则
**在 Tmux 内部的所有快捷键，都必须先按前缀键：**
👉 **`Ctrl + b`** （按住 Ctrl 和 b，**松手松开后**，再去按后面的指令键）

---

### 1. 终端命令（在 Tmux 外部/常规终端中输入）
*用于创建、查看和管理后台任务（Session）*

| 需求 | 完整命令 | 备注 |
| :--- | :--- | :--- |
| **新建并进入任务** | `tmux new -s <名字>` | 例：`tmux new -s qwen` |
| **新建静默后台任务** | `tmux new -s <名字> -d` | 创建后不进入，让它在后台默默待命 |
| **查看任务列表** | `tmux ls` | 列出当前所有在跑的 tmux 任务 |
| **进入(恢复)任务** | `tmux a -t <名字>` | ⚠️ **千万别漏了 `-t`**。例：`tmux a -t vm` |
| **关闭/终止指定任务** | `tmux kill-session -t <名字>` | 任务彻底被销毁，不再运行 |
| **一键终止所有任务** | `tmux kill-server` | 危险操作，清空所有 tmux 任务 |

---

### 2. 窗格操作 Pane（同屏切分显示）
*(前提：已在 tmux 内部，以下指令按完 `Ctrl + b` 松手后再按)*

| 快捷键 | 功能说明 |
| :---: | :--- |
| **`%`** | 左右切分出一个新窗格 |
| **`"`** | 上下切分出一个新窗格 |
| **`方向键`** | (上下左右) 切换光标到相邻窗格 |
| **`o`** | 在当前所有窗格之间轮换焦点 |
| **`z`** | **全屏放大**当前光标所在的窗格（再按一次恢复原状，看长日志神器） |
| **`x`** | 强制关闭当前光标所在的窗格（也可直接在命令行输入 `exit`） |

---

### 3. 窗口与高级操作 Window（类似浏览器的标签页）
*(前提：已在 tmux 内部，以下指令按完 `Ctrl + b` 松手后再按)*

| 快捷键 | 功能说明 |
| :---: | :--- |
| **`d`** | 🌟 **分离并挂起 (Detach)**：退回常规终端，Tmux 任务**继续在后台运行** |
| **`c`** | 新建一个全屏窗口（标签页） |
| **`p` / `n`** | 切换到 上一个(p) / 下一个(n) 窗口 |
| **`w`** | **开启“上帝视角”**：弹出会话和窗口列表，下方带**实时预览**（用方向键选择，回车进入，`q`退出） |
| **`$`** | 给当前任务（Session）重新命名 |
| **`&`** | 关闭当前整个窗口 |

### 💡 实用技巧与排错指南

1. **开启鼠标支持（强烈推荐）**
   * **操作**：按 `Ctrl + b` 后按冒号 **`:`**
   * **输入**：`set -g mouse on` 然后回车。
   * **效果**：你可以直接用鼠标点击切换窗格、拖拽改变窗格大小、使用鼠标滚轮上下翻看历史输出。
2. **终端卡死在 `[进程已完成]`**
   * **原因**：在 Mac 普通终端（非 tmux 内）输入了 `exit` 导致终端被杀死。
   * **解决**：按 `Cmd + W` 关掉废弃窗口，按 `Cmd + N` 重新打开一个新终端即可。后台任务不受影响。
3. **报错 `command attach-session: too many arguments`**
   * **原因**：进入任务时漏写了目标指定符。
   * **解决**：将 `tmux a vm` 改为完整的 `tmux a -t vm` 即可。
